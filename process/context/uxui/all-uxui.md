# UI/UX Context

This is the canonical UI/UX context entrypoint for cite-mind.

Use it after `process/context/all-context.md` when the task needs visual direction, interface conventions, or frontend experience guardrails.

## Scope

This group covers:

- current visual direction
- simplicity guardrails
- workspace UX priorities
- future design guidance for the app

It does not cover:

- backend implementation
- RAG internals
- testing commands

## Read When

Read this entrypoint when:

- redesigning screens
- changing navigation or workspace hierarchy
- deciding how much UI complexity to introduce
- aligning future frontend work with product intent

## Quick Routing

- use `process/context/uxui/uiux.md` for the current design direction and UX guardrails

## Source Paths

- `process/context/uxui/uiux.md`

## Update Triggers

Update this group when:

- the visual system changes materially
- new mature product surfaces appear
- navigation structure or simplicity rules change
